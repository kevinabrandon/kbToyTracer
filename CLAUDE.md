# kbToyTracer — notes for Claude

A Whitted-style ray tracer, recovered from a 2004/5 CD backup. James Arvo's
teaching kernel + Kevin Brandon's extensions; see PROVENANCE.md for whose code
is whose (`kb` prefix = Kevin's, unprefixed = Arvo's — keep that convention
when adding or renaming files).

## Build & run

    make                                   # -> ./kbtoytracer
    ./kbtoytracer scenes/foo.sdf           # writes foo.png next to the scene
    ./kbtoytracer scenes/foo.sdf --preview # fast 256x256 draft
    ./kbtoytracer scenes/foo.sdf --set aa_samples=4 --set seed=7

The scene decides *what* (lights, lens, materials); kbtoytracer.cfg decides
*how well* (resolution, `aa_samples`, `dof_samples`, `shadow_samples`,
`max_bounces`, `seed`). Use a nonzero `seed` for byte-reproducible renders.

Scenes support instancing: `define <name>` names an aggregate as a prototype,
`instance <name>` places it (shared geometry, per-instance transforms and
materials), and `import <file> as <name>` makes a prototype from another
.sdf/.sdf.gz scene (camera/lights skipped) or a Wavefront .obj/.obj.gz mesh
(basic .mtl: Kd/Ks/Ns/Ke). An instance given material lines repaints its
whole prototype; one without inherits the prototype's per-part materials.
Transforms compose in reading order: `matrix`, `translate`, `rotate`, `scale`.
Emitters are not allowed inside a define; the reader (kbReader.cpp) owns all
of this via its SceneReader. Stock meshes (Cornell box, Stanford dragon,
Lucy) live in scenes/models/ — see its README for provenance/licenses.

## Tests

    make test                        # golden-image regression suite (~8s)
    python3 tests/run_tests.py aa    # run only tests matching a term
    python3 tests/run_tests.py --bless   # re-generate goldens (only after a
                                         # deliberate, reviewed rendering change)

25 cases at 128x128 with seed=42 cover point/area lights, DoF, reflection,
refraction, transparent shadows, AA modes, textures, all three aggregates,
define/instance, and sdf/obj/mtl imports.
Renders must match tests/golden/ pixel-for-pixel; failures drop got/diff
images in tests/failures/. Run this after any change to src/. Goldens are
FP-sensitive, so bless them on the machine that runs the tests.

tests/index.html (regenerated by `--bless` or `run_tests.py --html`, never
hand-edited) shows every golden with what it tests and its overrides.

## Gallery

`python3 gen_all.py` re-renders the whole public gallery (gallery/index.html);
each gen_*.py script re-renders just its own section. Gallery configs pin
`shadow_samples=0` / `dof_samples=0` to keep the published images reproducible.
Don't commit re-rendered gallery images without the maintainer eyeballing them
first.
